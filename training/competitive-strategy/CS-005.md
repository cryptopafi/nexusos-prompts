# CS-005 — Map Sequential Game Trees with Backward Induction

## Version
2.0 — Opus v2 rewrite

## Problema
Many competitive interactions unfold sequentially — one firm moves first, then others respond. Analyzing these as simultaneous games produces incorrect predictions. Extensive-form game trees with backward induction reveal the subgame-perfect equilibrium: the rational outcome accounting for each player's ability to observe and respond to previous moves. This procedure builds and solves sequential games for strategic decision-making.

## Procedura
### Prerequisites
- Competitive scenario with clear move ordering (who acts first, second, etc.)
- Ability to identify decision nodes, information sets, and terminal payoffs
- CS-001 foundations (payoff estimation) applied to sequential context

### Steps
1. **Identify the sequence of moves** — Map who moves first, what they observe before moving, and what options they have at each decision point. Key question: is the first mover's choice observable before the second mover acts? If yes, it is a sequential game; if no, revert to simultaneous analysis (CS-001). Case: Market entry — incumbent observes entrant's decision before choosing to fight or accommodate.
2. **Draw the extensive-form game tree** — Start from the root (first mover's decision), branch for each option, then add subsequent decision nodes. Each branch = one possible action. Terminal nodes = final outcomes with payoffs for all players. Include Nature nodes for random events (demand uncertainty, regulatory outcomes) with associated probabilities.
   ```
   Root (P1) --> [Enter] --> P2 Node --> [Fight] --> (-10, -5)
                                    --> [Accommodate] --> (5, 10)
            --> [Stay Out] --> (0, 20)
   ```
3. **Assign payoffs at terminal nodes** — Estimate payoffs for every possible outcome path. Use NPV of future cash flows, not single-period profits. Apply Porter's Value Chain to estimate costs at each path. Validate with PESTLE: regulatory approval, technology readiness, and economic conditions affect terminal payoffs.
4. **Apply backward induction** — Start from the LAST decision nodes (closest to terminal). At each node, the deciding player chooses their payoff-maximizing option. Replace the decision node with the chosen outcome. Move backward through the tree, solving each node. The solution at the root = subgame-perfect equilibrium. Case: In the entry game above, P2 (incumbent) prefers Accommodate (10 > -5) at their node, so P1 knows entry leads to (5, 10) — which beats Stay Out (0) — so P1 enters.
5. **Identify subgame-perfect equilibrium (SPE)** — The SPE is the complete strategy profile surviving backward induction. It eliminates non-credible threats: promises or threats that the threatening player would not actually carry out at the relevant decision node. Critical insight: a threat to fight entry is credible ONLY if the incumbent prefers fighting to accommodating at that specific node.
6. **Test for commitment opportunities** — If the backward-induction outcome is unfavorable, evaluate whether first-mover or second-mover advantages can be created through commitments. Case: If the incumbent invests in excess capacity BEFORE the entrant decides, the fight payoff changes — making the threat credible and deterring entry. Link to CS-008 (Aggressive Commitment).
7. **Analyze information asymmetries** — If some moves are unobservable, convert those parts to simultaneous sub-games within the tree. If players have private information (costs, willingness to fight), use Bayesian game extensions — the uninformed player assigns probabilities to "types" and maximizes expected payoff. Case: Limit pricing (CS-017) works because the entrant cannot observe the incumbent's true costs.
8. **Extend to multi-stage games** — Many real competitive interactions have 3+ stages. Build the full tree and solve backward. For very large trees, use pruning: identify dominated branches early and eliminate them before full backward induction. Software tools (Gambit, game-theory solvers) help with trees exceeding 20 nodes.
9. **Translate SPE to strategic timeline** — Convert the equilibrium into an action plan with timing: "At T1, we do X. If competitor responds with Y, we do Z at T2." Include monitoring triggers at each decision node — what information do you need to observe before the next move? Create contingency branches for off-equilibrium play (rival does something irrational).
10. **Document the full game tree with solution** — Archive the tree diagram, backward induction path, SPE strategy profile, and key assumptions. This becomes the strategic playbook for the competitive interaction. Update when payoffs change or new moves become available.

### Verification
- Game tree includes all decision nodes, chance nodes, and terminal payoffs
- Backward induction applied correctly (last nodes solved first)
- SPE eliminates all non-credible threats
- Commitment opportunities evaluated for both players
- Strategic timeline maps directly to SPE path
- Off-equilibrium contingencies documented

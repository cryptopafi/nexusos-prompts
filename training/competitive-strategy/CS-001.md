# CS-001 — Construct a Competitive Game Matrix (Simultaneous Games)

## Version
2.0 — Opus v2 rewrite

## Problema
Competitive interactions where players move simultaneously (pricing, capacity, product launches) require structured payoff analysis to avoid reactive decision-making. Without a formal game matrix, firms misjudge rival responses, over-invest in dominated strategies, and miss cooperative equilibria. This procedure builds a complete normal-form game from scratch.

## Procedura
### Prerequisites
- Identified competitive interaction with 2+ players making simultaneous decisions
- Access to market data for payoff estimation (margins, market shares, historical moves)
- Familiarity with basic game theory notation (players, strategies, payoffs)

### Steps
1. **Define players and decision space** — List all relevant competitors (typically 2-3 for tractability). For each player, enumerate discrete strategy options (e.g., price high/low, enter/stay out, invest/hold). Case: Coca-Cola vs Pepsi advertising spend decisions — each chooses {heavy spend, moderate spend, minimal spend}.
2. **Estimate payoff values for each outcome combination** — For an NxM matrix (N strategies for Player 1, M for Player 2), calculate payoffs in all N*M cells. Use contribution margin, market share shift, or NPV. Apply Porter's Five Forces to contextualize: buyer power affects price-war payoffs, supplier power affects cost-based strategies. Document assumptions explicitly (e.g., "heavy ad spend = $50M, captures 3% share worth $120M revenue").
3. **Construct the normal-form matrix** — Build the payoff table with Player 1's strategies as rows, Player 2's as columns. Each cell contains (P1 payoff, P2 payoff). Validate symmetry: if the game should be symmetric (identical firms), verify mirrored payoffs. Template:
   ```
   |              | P2: Strategy A | P2: Strategy B |
   |P1: Strategy A| (π1_AA, π2_AA) | (π1_AB, π2_AB) |
   |P1: Strategy B| (π1_BA, π2_BA) | (π1_BB, π2_BB) |
   ```
4. **Test for dominated strategies** — For each player, check if any strategy yields lower payoffs than another across ALL opponent responses. Eliminate strictly dominated strategies iteratively (IESDS). This simplifies the matrix. Case: In the Bertrand duopoly, pricing above competitor's price is dominated when products are homogeneous.
5. **Identify Nash Equilibria** — For each cell, check if either player would unilaterally deviate. A cell is a Nash Equilibrium if neither player improves by switching alone. Mark all NE. Multiple NE may exist (coordination problem) or none in pure strategies (requiring mixed strategy analysis).
6. **Assess payoff sensitivity** — Vary key assumptions by +/-20% and recheck NE stability. If small changes shift equilibria, the game outcome is fragile and requires scenario planning. Use PESTLE factors (regulatory changes, technology shifts) as sensitivity drivers.
7. **Evaluate cooperative vs. non-cooperative outcomes** — Compare NE payoffs to the joint-maximizing outcome. The gap represents value destroyed by competition (deadweight loss of rivalry). Assess whether tacit coordination is feasible given market structure, transparency, and antitrust constraints.
8. **Map to strategic recommendations** — Translate matrix insights into action: commit to dominant strategy if one exists, signal intentions if coordination is valuable, or invest in changing the game structure (adding strategies, changing payoffs through differentiation). Document the recommended strategy with contingency plans.
9. **Create a monitoring trigger matrix** — Define observable signals that indicate rivals have shifted strategies (price changes, capacity announcements, patent filings). Link each signal to a re-analysis trigger. Update the game matrix quarterly or upon trigger activation.

### Verification
- Matrix has correct dimensions (all strategy combinations covered)
- Payoffs are internally consistent (zero-sum check or value-creation check)
- At least one Nash Equilibrium identified or mixed-strategy analysis performed
- Sensitivity analysis covers 3+ key variables
- Strategic recommendation ties directly to equilibrium analysis

# BS-016 — Implement Metropolis-Hastings Algorithm

## Problema
Most real Bayesian posteriors are non-conjugate and require MCMC sampling for inference.

## Procedura
### Prerequisites
- R or Python
- BS-015 completed

### Steps
1. Write the target posterior (unnormalized) as a function.
2. Choose a proposal distribution: symmetric random walk N(current, sigma).
3. Implement the MH loop: propose, compute acceptance ratio, accept/reject.
4. Run chain for N iterations with burn-in period.
5. Tune proposal variance: aim for 20-40% acceptance rate.
6. Compute posterior summaries from the chain (discarding burn-in).
7. Plot trace plot and histogram of samples.

### Verification
- Chain mixes well (no stuck periods in trace plot).
- Posterior summaries match known results for test cases.

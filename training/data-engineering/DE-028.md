# DE-028 — Write Sub-Queries and Nested SELECTs

## Problema
Sub-queries enable complex data retrieval that cannot be achieved with simple joins or single-level queries.

## Procedura
### Prerequisites
- Database with related tables

### Steps
1. Write a sub-query in the WHERE clause
2. Write a sub-query in the FROM clause (derived table)
3. Write a sub-query in the SELECT clause (scalar sub-query)
4. Use EXISTS and NOT EXISTS with correlated sub-queries
5. Use IN and NOT IN with sub-queries
6. Compare sub-query performance vs JOIN alternatives
7. Rewrite complex sub-queries as CTEs for readability

### Verification
- Sub-queries return correct results
- CTEs improve readability of complex queries

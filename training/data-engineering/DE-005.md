# DE-005 — Implement Data Governance and Compliance Frameworks

**Version**: 2.0
**Domain**: data-engineering
**Level**: ADVANCED
**FORGE**: PASS

## Problema
Without formal governance, organizations face regulatory fines (GDPR up to 4% of global revenue), data inconsistencies across teams (30%+ of enterprise data is duplicate or contradictory), and inability to answer "where did this number come from?" — destroying trust in data-driven decisions.

## Procedura
### Prerequisites
- Data catalog tool access (DataHub, Atlan, or AWS Glue Data Catalog)
- Understanding of applicable regulations (GDPR, CCPA, HIPAA, SOX)
- Stakeholder alignment on data ownership model

### Steps
1. **Establish a data classification framework** — Define 4 tiers: Public (open data), Internal (business metrics), Confidential (PII, financial), Restricted (PHI, payment). Tag every table/column in the data catalog. Use automated classifiers (AWS Macie, Google DLP, Presidio) for PII detection. Benchmark: 100% of production tables classified within 30 days.

2. **Define data ownership and stewardship** — Assign for each domain: Data Owner (business leader, approves access), Data Steward (maintains quality rules), Data Engineer (implements pipelines). Document in a RACI matrix. In Snowflake: map owners to `OWNERSHIP` grants. In dbt: use `meta.owner` in schema.yml.

3. **Implement a data catalog with lineage** — Deploy DataHub or Atlan. Ingest metadata from: databases (Snowflake, BigQuery, PostgreSQL), transformation tools (dbt, Spark), orchestrators (Airflow). Enable column-level lineage tracking. In dbt: use `dbt docs generate` + `dbt source freshness` for automated documentation. Benchmark: lineage should trace any metric back to raw source in <5 clicks.

4. **Build data quality SLAs per tier** — Define rules: Tier 1 (revenue-critical) = 99.9% completeness, <15min freshness, <0.1% error rate. Tier 2 (operational) = 99% completeness, <1hr freshness. Tier 3 (analytical) = 95% completeness, <24hr freshness. Implement with Great Expectations, dbt tests, or Monte Carlo. Alert owners on SLA breach within 5 minutes.

5. **Implement access control policies** — Design role-based access: Analyst (read gold layer), Engineer (read/write silver/gold), Admin (full access). In Snowflake: use managed access schemas + dynamic data masking for PII. In BigQuery: use authorized views to expose only non-sensitive columns. Audit: run quarterly access reviews, revoke unused permissions (>90 days inactive).

6. **Set up data lineage and impact analysis** — Configure end-to-end lineage: source → ingestion → transformation → serving → dashboard. In dbt: use `ref()` and `source()` for automatic lineage. In Airflow: use dataset-aware scheduling. Before any schema change, run impact analysis: "which dashboards break if I rename this column?" Use DataHub or dbt `ls --select +model_name+`.

7. **Implement compliance automation** — For GDPR: build a `data_subject_delete(user_id)` pipeline that cascades deletions across all tables within 30 days. For data retention: schedule automatic purging of data beyond retention period (Snowflake `DATA_RETENTION_TIME_IN_DAYS`, BigQuery table expiration). Log all deletion/retention actions for audit trail.

8. **Create a data contract framework** — Define contracts between producers and consumers: schema (column names, types, constraints), SLAs (freshness, completeness), semantic rules (business logic). Implement with dbt contracts (`constraints`, `data_type` enforcement in v1.5+) or Soda. Break builds when contracts are violated.

9. **Build governance dashboards and reporting** — Track: data quality scores per domain, SLA compliance rates, access review completion, PII exposure incidents, lineage coverage percentage. Report monthly to data council. Use Looker/Tableau connected to governance metadata tables. Target: >95% quality SLA compliance, 100% classification coverage.

### Verification
- 100% of production tables classified and cataloged with owners assigned
- Data lineage traces any metric to raw source within 5 clicks
- GDPR deletion pipeline tested and completes within 30-day window
- Governance dashboard shows >95% quality SLA compliance
- Quarterly access reviews documented with unused permissions revoked

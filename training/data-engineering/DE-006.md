# DE-006 — Set Up Python with Jupyter Notebooks for Data Engineering

**Version**: 2.0
**Domain**: data-engineering
**Level**: BEGINNER
**FORGE**: PASS

## Problema
Data engineers waste hours debugging environment issues — conflicting package versions, missing dependencies, and irreproducible notebooks. A properly configured Python+Jupyter environment eliminates "works on my machine" failures and accelerates prototyping-to-production workflows.

## Procedura
### Prerequisites
- Machine with Python 3.9+ or access to cloud notebooks (GCP Vertex AI, AWS SageMaker, Databricks)
- Basic command-line familiarity

### Steps
1. **Install Python with virtual environment isolation** — Use `pyenv` for Python version management: `pyenv install 3.11.7 && pyenv global 3.11.7`. Create project-specific environments with `python -m venv .venv` or `conda create -n de-env python=3.11`. Never install packages in system Python. Benchmark: environment creation should take <2min.

2. **Install core data engineering packages** — Install in order (dependency-aware): `pip install pandas==2.1.4 numpy==1.26.2 pyarrow==14.0.1 sqlalchemy==2.0.23 psycopg2-binary==2.9.9 jupyter==1.0.0 jupyterlab==4.0.9`. Pin versions in `requirements.txt`. For heavy workloads add: `polars`, `dask`, `pyspark`. Total install size: ~500MB for core stack.

3. **Configure JupyterLab for data engineering workflows** — Install extensions: `jupyterlab-git` (version control), `jupyterlab-sql` (database connections), `jupyterlab-lsp` (autocomplete). Configure memory limits: `jupyter lab --NotebookApp.max_buffer_size=1073741824` (1GB for large datasets). Set autosave to 120s. Create a startup script in `~/.ipython/profile_default/startup/00-imports.py` with common imports.

4. **Set up database connectivity** — Configure SQLAlchemy connections for common databases: PostgreSQL (`postgresql://user:pass@host:5432/db`), MySQL (`mysql+pymysql://`), Snowflake (`snowflake://user:pass@account/db/schema`), BigQuery (`bigquery://project`). Store credentials in `.env` file, load with `python-dotenv`. Never hardcode credentials in notebooks.

5. **Configure notebook-to-script conversion** — Install `nbconvert` and `jupytext`. Use `jupytext --set-formats ipynb,py:percent` to auto-sync notebooks to .py files (version-control friendly). Set up pre-commit hook: `nbstripout` to remove outputs before committing. This ensures notebooks are reproducible and reviewable in PRs.

6. **Set up data profiling and visualization** — Install `ydata-profiling` for automatic EDA reports, `matplotlib`+`seaborn` for custom plots, `plotly` for interactive dashboards. Configure default plot sizes: `plt.rcParams['figure.figsize'] = [12, 6]`. For large datasets: use `datashader` or `hvplot` (renders millions of points in <1s).

7. **Create a project template structure** — Standardize directory layout: `notebooks/` (exploration), `src/` (production code), `tests/` (pytest), `data/raw/` (immutable inputs), `data/processed/` (outputs), `configs/` (YAML configs), `.env` (credentials, gitignored). Use `cookiecutter-data-science` template or create custom. Include `Makefile` with targets: `setup`, `test`, `lint`, `run`.

8. **Configure linting and code quality** — Install `ruff` (replaces flake8+isort+black, 10-100x faster), `mypy` for type checking. Configure in `pyproject.toml`: line length 120, target Python 3.11. Set up pre-commit hooks: `ruff check`, `ruff format`, `mypy`. Run on every commit to maintain code quality from notebook prototyping to production.

### Verification
- Python environment activates cleanly with all packages importable
- JupyterLab launches with extensions functional (git, SQL, LSP)
- Database connections work from notebook (test query returns results)
- Notebook-to-script conversion produces clean, runnable .py files
- Pre-commit hooks pass (linting, output stripping)

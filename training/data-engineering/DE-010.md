# DE-010 — Perform NumPy Array Operations for Data Engineering

**Version**: 2.0
**Domain**: data-engineering
**Level**: INTERMEDIATE
**FORGE**: PASS

## Problema
Python loops over millions of data points take minutes; NumPy vectorized operations complete the same work in milliseconds. Data engineers who skip NumPy fundamentals write pipelines that are 10-1000x slower than necessary, hitting timeout SLAs on transforms that should be trivial.

## Procedura
### Prerequisites
- Python 3.9+ with numpy 1.26+ installed
- Understanding of basic linear algebra concepts
- Jupyter notebook for interactive benchmarking

### Steps
1. **Create arrays from pipeline data sources** — Convert lists/dicts to arrays: `np.array(data, dtype=np.float32)` (specify dtype to control memory). Use `np.fromiter(generator, dtype, count)` for streaming data. Create structured arrays for tabular data: `np.array(records, dtype=[('id', 'i4'), ('amount', 'f8')])`. Benchmark: array creation from 1M-element list takes ~50ms.

2. **Master vectorized arithmetic for transforms** — Replace loops with vectorized ops: `prices * 1.1` (element-wise), `np.dot(weights, features)` (matrix multiply for scoring). Use `np.where(condition, true_val, false_val)` for conditional transforms. Chain operations: `np.clip(np.log1p(values), 0, 10)`. Benchmark: vectorized multiply on 10M elements takes ~15ms vs ~3s for Python loop (200x speedup).

3. **Apply aggregations for pipeline statistics** — Compute: `np.mean(arr, axis=0)` (column means), `np.percentile(arr, [25, 50, 75, 95, 99])` for distribution analysis, `np.nanmean()` for null-safe aggregation. Use `np.histogram(arr, bins=50)` for distribution profiling. For grouped aggregations: use `np.bincount(labels, weights=values)` (faster than pandas groupby for simple cases).

4. **Use broadcasting for multi-dimensional transforms** — Normalize columns: `(data - data.mean(axis=0)) / data.std(axis=0)` (z-score, broadcasts automatically). Add seasonal adjustments: `daily_data + monthly_factors[:, np.newaxis]`. Rules: dimensions align from right, size-1 dimensions stretch. Broadcasting avoids creating temporary arrays (saves 50%+ memory).

5. **Optimize memory with dtype selection and views** — Choose minimal dtype: `float32` vs `float64` (50% memory saving, sufficient for most analytics), `int16` for small integers. Use views instead of copies: `arr[::2]` creates a view (zero memory). Check: `arr.flags['OWNDATA']` to verify view vs copy. For boolean masks: `np.packbits()` compresses 8x. Benchmark: 100M float32 = 400MB vs float64 = 800MB.

6. **Handle missing data with masked arrays** — Use `np.ma.masked_invalid(arr)` to mask NaN/inf. Apply operations: `np.ma.mean(masked_arr)` ignores masked values. For pipeline use: `np.nan_to_num(arr, nan=0.0, posinf=1e10, neginf=-1e10)` for safe downstream processing. Prefer `np.nansum()`, `np.nanmean()` over masked arrays for simple cases (2-3x faster).

7. **Implement efficient sorting and searching** — Use `np.argsort()` for sort-order indices (preserves original array). `np.searchsorted(sorted_arr, values)` for binary search insertion points (O(log n)). `np.unique(arr, return_counts=True)` for value counting. For top-N: `np.argpartition(arr, -N)[-N:]` is O(n) vs O(n log n) for full sort. Benchmark: argpartition on 10M elements takes ~30ms vs ~800ms for argsort.

8. **Use NumPy for data quality checks** — Detect anomalies: `np.abs(arr - np.median(arr)) > 3 * np.std(arr)` (3-sigma outliers). Check monotonicity: `np.all(np.diff(timestamps) > 0)`. Validate ranges: `np.all((arr >= 0) & (arr <= 1000))`. Count issues: `np.sum(np.isnan(arr))`, `np.sum(np.isinf(arr))`. Generate quality report as dict of metrics.

9. **Bridge NumPy to pipeline frameworks** — Convert to pandas: `pd.DataFrame(arr, columns=names)` (zero-copy with same dtypes). Convert to Spark: use `pyarrow` as intermediary (`pa.array(np_arr)`). Save to Parquet: `pa.Table.from_arrays([pa.array(col) for col in arr.T], names=columns)`. For GPU acceleration: `cupy.array(np_arr)` provides identical API with 10-100x speedup on NVIDIA GPUs.

### Verification
- Vectorized operations achieve >100x speedup over Python loops (timeit verified)
- Memory usage optimized with appropriate dtypes (float32 where applicable)
- Broadcasting used for multi-dimensional transforms without temporary arrays
- Data quality checks detect injected anomalies in test dataset
- NumPy arrays successfully convert to pandas/pyarrow for pipeline handoff

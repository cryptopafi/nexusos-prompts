# DE-053 — Design a Snowflake Schema Data Warehouse

## Problema
Snowflake schemas normalize dimensions for storage efficiency, trading query simplicity for reduced redundancy.

## Procedura
### Prerequisites
- Star schema experience (DE-052)
- Data warehouse access

### Steps
1. Start with a star schema design (DE-052)
2. Identify dimensions with hierarchical attributes
3. Normalize hierarchical dimensions into sub-dimension tables
4. Create foreign keys between dimension levels
5. Compare query complexity: snowflake vs star
6. Document when snowflake schema is preferred over star
7. Implement and validate with analytical queries

### Verification
- Snowflake schema is properly normalized
- Queries produce correct results across dimension hierarchies

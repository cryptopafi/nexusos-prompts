# DE-064 — Design NoSQL Data Models

## Problema
Applying relational thinking to NoSQL databases leads to poor performance; query-driven design is essential.

## Procedura
### Prerequisites
- Understanding of target NoSQL database (MongoDB or Cassandra)

### Steps
1. Identify the application's query patterns first
2. Design tables around query patterns (not entities)
3. Choose partition keys for even data distribution
4. Choose clustering columns for query sorting needs
5. Denormalize data to avoid joins
6. Handle one-to-many relationships with collections or separate tables
7. Validate the model with expected query workloads

### Verification
- Data model supports all required queries efficiently
- No full-table scans are required for common queries

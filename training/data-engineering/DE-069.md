# DE-069 — Monitor and Tune Spark Applications

## Problema
Production Spark jobs need monitoring to detect performance degradation, data skew, and resource contention.

## Procedura
### Prerequisites
- Production Spark cluster
- Spark History Server configured

### Steps
1. Access the Spark History Server UI
2. Analyze job stages and identify bottlenecks
3. Detect data skew from task duration distributions
4. Identify spill-to-disk events indicating insufficient memory
5. Monitor shuffle read/write volumes
6. Set up alerts for job failures and duration thresholds
7. Implement adaptive query execution (AQE) for Spark 3.x

### Verification
- Bottlenecks are identified and documented
- Tuning improvements are measured with before/after metrics

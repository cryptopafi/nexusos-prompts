# DE-023 — Create Views in PostgreSQL

## Problema
Views simplify complex queries, enforce security by limiting data access, and provide stable interfaces for applications.

## Procedura
### Prerequisites
- PostgreSQL with tables and data (DE-022)

### Steps
1. Create a simple view from a SELECT statement
2. Create a view joining multiple tables
3. Create a materialized view for performance
4. Refresh a materialized view with REFRESH MATERIALIZED VIEW
5. Grant and revoke access on views for security
6. Drop and recreate views for schema changes

### Verification
- Views return correct data matching the underlying query
- Materialized views refresh successfully

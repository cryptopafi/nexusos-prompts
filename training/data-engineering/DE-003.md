# DE-003 ŌĆö Implement Data Wrangling with Python Tools

**Version**: 2.0
**Domain**: data-engineering
**Level**: INTERMEDIATE
**FORGE**: PASS

## Problema
Raw data from production sources is messy ŌĆö 60-80% of a data engineer's time goes to wrangling (type mismatches, missing values, inconsistent formats, nested JSON). Without systematic wrangling patterns, pipelines break silently and downstream analytics produce wrong results.

## Procedura
### Prerequisites
- Python 3.9+ with pandas, numpy, and pyarrow installed
- Sample dataset with known quality issues (nulls, duplicates, mixed types)
- Jupyter notebook or IDE environment

### Steps
1. **Profile the raw dataset** ŌĆö Use `ydata-profiling` (formerly pandas-profiling) or `whylogs` to generate a data quality report. Identify: missing value percentages per column, data type mismatches, cardinality of categorical columns, outlier distributions. Benchmark: profiling 1M rows completes in <30s with ydata-profiling.

2. **Standardize data types and formats** ŌĆö Convert columns to appropriate types: `pd.to_datetime()` for dates (handle mixed formats with `infer_datetime_format=True`), `pd.to_numeric(errors='coerce')` for numbers with string contamination, `.astype('category')` for low-cardinality columns (saves 50-90% memory). Parse nested JSON with `pd.json_normalize(record_path, meta)`.

3. **Handle missing values with strategy per column** ŌĆö Apply context-appropriate imputation: forward-fill for time-series (`ffill`), median for numeric columns with outliers, mode for categoricals, or flag-and-keep for analytical columns. Drop rows only when missingness is <5% and MCAR. Document the strategy in a data dictionary.

4. **Deduplicate with composite keys** ŌĆö Identify duplicates using `df.duplicated(subset=['key1','key2'], keep='first')`. For fuzzy duplicates: use `recordlinkage` or `thefuzz` with threshold >85% similarity. Benchmark: exact dedup on 10M rows completes in <5s with pandas; use Spark for >100M rows.

5. **Normalize text and categorical data** ŌĆö Standardize: `.str.strip().str.lower()` for text, map abbreviations to full forms, validate against reference datasets. Use regex for phone/email/postal validation. Apply `unidecode` for accent normalization in multilingual data.

6. **Engineer derived features** ŌĆö Create: date parts (year, month, day_of_week, is_weekend), binned numerics (`pd.cut`/`pd.qcut`), rolling aggregates (7d/30d windows), lag features for time-series. Name with prefix convention: `derived_`, `flag_`, `bin_`.

7. **Validate output with Great Expectations** ŌĆö Define expectation suite: `expect_column_values_to_not_be_null`, `expect_column_values_to_be_between`, `expect_column_values_to_be_in_set`. Generate data docs HTML report. Fail pipeline on critical expectation failures.

8. **Optimize memory and performance** ŌĆö Downcast numerics (`pd.to_numeric(downcast='integer')`), use `category` dtype for <50 unique values, read large files in chunks (`chunksize=100_000`). Switch to `polars` for 2-5x speedup on single-machine wrangling. For >10GB: use PySpark or Dask.

9. **Package as reusable wrangling module** ŌĆö Create `wrangling.py` with typed functions: `standardize_types(df, schema)`, `handle_nulls(df, strategy_map)`, `deduplicate(df, keys)`, `validate(df, suite)`. Include docstrings, type hints, and pytest unit tests with >80% coverage.

### Verification
- Data profiling report shows 0% type mismatches post-wrangling
- Great Expectations suite passes with 100% success rate
- Memory usage reduced by >30% through type optimization
- Wrangling functions have unit tests with >80% coverage

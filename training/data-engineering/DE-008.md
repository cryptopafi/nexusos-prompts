# DE-008 — Read and Write Files with Python for Data Pipelines

**Version**: 2.0
**Domain**: data-engineering
**Level**: BEGINNER
**FORGE**: PASS

## Problema
Data pipelines ingest files in dozens of formats (CSV, JSON, Parquet, Avro, XML). Wrong encoding handling corrupts data silently, unbuffered reads cause OOM on large files, and missing error handling creates partial writes that pollute downstream tables.

## Procedura
### Prerequisites
- Python 3.9+ with pyarrow, orjson installed
- Sample files in CSV, JSON, Parquet formats (1MB-1GB range)

### Steps
1. **Read CSV files with production-grade settings** — Use `pd.read_csv()` with explicit parameters: `dtype` dict (prevent type inference surprises), `parse_dates=['date_col']`, `encoding='utf-8'` (try `latin-1` as fallback), `na_values=['', 'NULL', 'N/A']`, `chunksize=100_000` for files >500MB. For maximum speed: `polars.read_csv()` is 5-10x faster than pandas. Benchmark: pandas reads 1GB CSV in ~15s, polars in ~2s.

2. **Handle JSON files and NDJSON streams** — For single JSON: `json.load(f)` with `orjson.loads()` (3-5x faster). For NDJSON (newline-delimited, common in API exports): `pd.read_json(path, lines=True)`. For large JSON arrays: use `ijson` for streaming parse (constant memory). Handle encoding: `open(path, encoding='utf-8-sig')` to skip BOM. Validate structure before full parse with `json.JSONDecodeError` handling.

3. **Use Parquet as the default interchange format** — Write: `df.to_parquet(path, engine='pyarrow', compression='snappy')`. Snappy = fastest decompression (3GB/s), gzip = smallest size (2-3x compression). Use partitioning for large datasets: `df.to_parquet(path, partition_cols=['year', 'month'])`. Read specific columns: `pd.read_parquet(path, columns=['id', 'amount'])` — reads only needed columns (columnar advantage). Benchmark: Parquet is 2-10x smaller than CSV and 5-50x faster to read.

4. **Implement atomic file writes** — Never write directly to the target path (partial writes on failure). Pattern: write to `{target}.tmp`, then `os.rename('{target}.tmp', target)` (atomic on same filesystem). For cloud: use `smart_open` library for streaming to S3/GCS. Implement checksum verification: `hashlib.md5(content).hexdigest()` written alongside data file.

5. **Handle large files with streaming patterns** — For files >1GB: use chunked reading (`chunksize` in pandas, `batch_size` in pyarrow). Process with generator pipeline: `read_chunk → transform → write_chunk`. For Parquet: use `pyarrow.parquet.ParquetFile` with `read_row_group()` for parallel processing. Memory target: process any file size in <500MB RAM.

6. **Read from and write to cloud storage** — Configure `fsspec` with `s3fs` (AWS), `gcsfs` (GCP), `adlfs` (Azure). Read directly: `pd.read_parquet('s3://bucket/path.parquet')`. Use `smart_open` for streaming: `with smart_open.open('s3://bucket/file.gz', 'rb') as f`. Set retry policies: 3 retries with exponential backoff (1s, 2s, 4s). Configure multipart upload for files >100MB.

7. **Implement file format conversion pipelines** — Build converters: CSV→Parquet (with schema enforcement), JSON→Parquet (with `pd.json_normalize` for nested), XML→Parquet (with `lxml` or `xmltodict`). Add schema validation at conversion: verify column count, types, and nullable constraints. Log conversion metrics: input size, output size, compression ratio, row count, duration.

8. **Handle encoding and character set issues** — Detect encoding with `chardet` or `charset_normalizer` (faster). Common pipeline: detect → decode → normalize → encode as UTF-8. Handle mixed encodings in same file (common in legacy exports): read binary, detect per-line if needed. Strip null bytes: `content.replace(b'\x00', b'')`. Log encoding issues for source system feedback.

### Verification
- Can read/write CSV, JSON, Parquet with explicit encoding and type handling
- Atomic write pattern prevents partial file creation on failure
- Large file processing (>1GB) stays within 500MB memory budget
- Cloud storage read/write works with retry and error handling
- File conversion pipeline produces valid Parquet with schema enforcement

# DE-018 — Design Primary Keys, Foreign Keys, and Indexes

## Problema
Proper key and index design ensures data integrity and query performance in relational databases.

## Procedura
### Prerequisites
- Database with tables created (DE-017)

### Steps
1. Choose appropriate primary keys (natural vs surrogate)
2. Define foreign keys to enforce referential integrity
3. Create unique indexes for business-unique columns
4. Create composite indexes for multi-column queries
5. Analyze query patterns to determine which indexes to create
6. Test index effectiveness with EXPLAIN
7. Document the indexing strategy

### Verification
- Primary and foreign keys enforce data integrity
- Indexes improve query performance (verified with EXPLAIN)

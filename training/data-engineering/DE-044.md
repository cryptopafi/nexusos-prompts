# DE-044 — Monitor Database Performance and Optimize Queries

## Problema
Unmonitored databases degrade silently until they cause outages; proactive monitoring prevents production incidents.

## Procedura
### Prerequisites
- Database admin access
- Monitoring tool (pg_stat_statements, Performance Schema)

### Steps
1. Set up monitoring for key metrics: QPS, latency, connections, disk I/O
2. Enable slow query logging
3. Use EXPLAIN ANALYZE to diagnose slow queries
4. Identify and optimize the top 5 slowest queries
5. Monitor and manage table bloat and fragmentation
6. Set up alerts for critical thresholds
7. Create a weekly performance review process

### Verification
- Monitoring dashboard shows key metrics
- Top slow queries are identified and optimized
- Alerts are configured and tested

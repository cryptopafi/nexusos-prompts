# DE-019 — Normalize Database Tables (1NF, 2NF, 3NF)

## Problema
Unnormalized tables cause data redundancy, update anomalies, and inconsistencies that corrupt data over time.

## Procedura
### Prerequisites
- Understanding of functional dependencies
- Sample denormalized data

### Steps
1. Identify a denormalized table with redundant data
2. Apply 1NF: ensure atomic values and no repeating groups
3. Apply 2NF: remove partial dependencies on composite keys
4. Apply 3NF: remove transitive dependencies
5. Verify each normal form with test data
6. Document trade-offs of normalization vs performance
7. Know when to denormalize intentionally

### Verification
- Tables meet 3NF requirements
- No update anomalies exist in the normalized schema

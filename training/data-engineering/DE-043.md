# DE-043 — Manage Database Users, Roles, and Permissions

## Problema
Improper access control exposes data to unauthorized users and violates compliance requirements.

## Procedura
### Prerequisites
- Database admin access

### Steps
1. Create database users for different team members
2. Create roles for common permission sets (read-only, read-write, admin)
3. Grant appropriate privileges to roles
4. Assign roles to users
5. Revoke permissions when access is no longer needed
6. Implement password policies
7. Audit user access and permissions regularly

### Verification
- Users can only access data within their permissions
- Role-based access is consistent and documented

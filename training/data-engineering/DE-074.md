# DE-074 — Design OLTP Data Platform Architecture

## Problema
OLTP systems require high availability, low latency, and ACID compliance; architecture must balance these requirements.

## Procedura
### Prerequisites
- Business transaction requirements
- Cloud or on-premise infrastructure

### Steps
1. Define transaction volume and latency requirements
2. Select an OLTP database (PostgreSQL, MySQL, CockroachDB)
3. Design for high availability: replication and failover
4. Implement connection pooling for efficient resource use
5. Design the schema for write-optimized operations
6. Plan for scaling: read replicas, sharding
7. Document the architecture with SLA targets

### Verification
- Architecture meets latency and availability requirements
- Failover procedure is documented and tested

# DE-031 — Implement ACID Transactions in SQL

## Problema
Without proper transactions, concurrent data modifications can leave the database in an inconsistent state.

## Procedura
### Prerequisites
- Database access with transaction support

### Steps
1. Understand ACID properties: Atomicity, Consistency, Isolation, Durability
2. Begin a transaction with BEGIN or START TRANSACTION
3. Execute multiple DML statements within the transaction
4. Commit the transaction with COMMIT
5. Rollback the transaction with ROLLBACK on error
6. Implement savepoints for partial rollbacks
7. Test concurrent transactions to observe isolation behavior

### Verification
- Committed transactions persist across sessions
- Rolled-back transactions leave no changes
- Concurrent access is handled correctly

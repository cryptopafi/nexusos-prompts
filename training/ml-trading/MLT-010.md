# MLT-010 — Design Backtesting Frameworks for Trading Strategies

## Problema
Without rigorous backtesting, trading strategies may seem profitable but fail due to lookahead bias or survivorship bias.

## Procedura
### Prerequisites
- Historical price data
- Python with backtrader or custom framework

### Steps
1. Define the strategy rules (entry, exit, position sizing)
2. Implement a backtesting engine with walk-forward validation
3. Ensure no lookahead bias: only use data available at decision time
4. Account for transaction costs, slippage, and market impact
5. Calculate key metrics: Sharpe ratio, max drawdown, win rate
6. Compare against benchmark (buy-and-hold, random)
7. Document all assumptions and limitations

### Verification
- Backtest produces realistic performance metrics
- No lookahead bias confirmed by code review
- Sharpe ratio and drawdown are documented

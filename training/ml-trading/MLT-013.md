# MLT-013 — Build Quantitative Trading Strategies (Entries/Exits)

## Problema
Ad-hoc trading decisions lead to inconsistent results; systematic entry/exit rules enforce discipline and enable testing.

## Procedura
### Prerequisites
- Historical market data
- Python with trading libraries

### Steps
1. Define the market hypothesis your strategy exploits
2. Code entry signals based on technical indicators or ML predictions
3. Code exit signals: stop-loss, take-profit, time-based, signal-based
4. Implement position sizing rules (fixed, percent, Kelly criterion)
5. Backtest on historical data with walk-forward validation
6. Optimize parameters without overfitting (cross-validation)
7. Document the strategy with all rules and rationale

### Verification
- Strategy has clear, codified entry and exit rules
- Backtest results show strategy logic executes correctly

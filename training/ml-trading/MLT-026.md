# MLT-026 — Backtest Pair Trading Strategy and Avoid Overfitting

## Problema
Pair trading backtests are especially prone to overfitting because of multiple degrees of freedom in pair selection and thresholds.

## Procedura
### Prerequisites
- Historical data spanning multiple years
- Pair trading implementation (MLT-023/024)

### Steps
1. Use walk-forward optimization instead of single backtest
2. Split data into multiple train-test windows chronologically
3. In each window: select pairs, fit parameters, test on next period
4. Avoid optimizing entry/exit z-score thresholds on full dataset
5. Use Bonferroni correction for multiple pair comparisons
6. Report aggregate performance across all walk-forward windows
7. Compare to a null hypothesis of random pair selection

### Verification
- Walk-forward results are consistent across windows
- Performance survives multiple comparison corrections

# MLT-018 — Serve ML Models in the Cloud for Trading Predictions

## Problema
Trained models sitting on a laptop cannot serve real-time trading predictions; cloud deployment enables live usage.

## Procedura
### Prerequisites
- Trained ML model
- Cloud account (GCP, AWS, or self-hosted)

### Steps
1. Export the trained model (SavedModel format for TF, pickle for sklearn)
2. Set up a cloud serving endpoint (GCP AI Platform, AWS SageMaker, or FastAPI)
3. Upload the model to the serving infrastructure
4. Create a prediction API endpoint
5. Test the endpoint with sample trading data
6. Implement monitoring for prediction latency and drift
7. Set up automated model retraining pipeline

### Verification
- API endpoint returns predictions within acceptable latency
- Monitoring dashboard shows prediction metrics

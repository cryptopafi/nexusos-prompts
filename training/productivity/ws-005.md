# WS-005 — Estimate Task Duration Accurately

## Problema
Chronic underestimation of task duration (planning fallacy) causes cascading schedule failures and stress.

## Procedura
### Prerequisites
- Historical task data or willingness to start tracking

### Steps
1. Record actual time taken for tasks over 2 weeks (create a reference class database)
2. Compare estimated vs actual duration for each task to calculate your personal bias factor
3. Apply the reference class method: use actual past data for similar tasks instead of optimistic estimates
4. Add buffer multiplier based on task uncertainty: 1.5x for familiar tasks, 2-3x for novel tasks
5. Break complex tasks into subtasks and estimate each separately (reduce scope for each estimate)
6. Use pre-mortem technique: imagine the task took twice as long, identify what went wrong
7. Track estimation accuracy over time and adjust multipliers

### Verification
- Estimation accuracy within 20% for at least 70% of tasks after 4 weeks of practice
- Reference class database contains at least 20 task entries

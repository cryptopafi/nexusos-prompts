# VC-011 — Understand Model Context Protocol (MCP)

## Problema
Developers treat AI coding tools as isolated chatbots, unaware that MCP enables AI to interact with external tools (databases, APIs, file systems, cloud services) — limiting AI to code generation only.

## Procedura
### Prerequisites
- Basic understanding of API protocols (REST, JSON-RPC)
- Familiarity with at least one AI coding tool (Cursor, Copilot, Claude Code)

### Steps
1. **Understand what MCP solves** — MCP (Model Context Protocol) is an open standard by Anthropic that connects AI models to external tools and data sources. Without MCP, AI can only read/write code. With MCP, AI can: query databases, call APIs, read docs, manage infrastructure, access file systems.
2. **Learn the architecture** — MCP uses a client-server model. The AI tool (Cursor, Claude Code) is the **client**. External tools run as **MCP servers**. Communication uses JSON-RPC over stdio or HTTP/SSE. The client discovers server capabilities via a `tools/list` call.
3. **Understand the three primitives** — MCP servers expose: (a) **Tools** — functions the AI can call (e.g., `query_database`, `create_file`), (b) **Resources** — data the AI can read (e.g., database schemas, API docs), (c) **Prompts** — pre-built prompt templates for common tasks.
4. **Explore the MCP ecosystem** — visit `github.com/modelcontextprotocol` for the spec and official servers. Popular MCP servers: filesystem (file operations), GitHub (repo management), PostgreSQL (database queries), Brave Search (web search), Puppeteer (browser automation).
5. **Understand the security model** — MCP servers run locally or on trusted infrastructure. The AI client requests tool calls; the user approves or auto-approves. Sensitive operations (file writes, API calls) should require explicit approval. Never run untrusted MCP servers.
6. **Compare MCP to alternatives** — vs function calling: MCP is standardized and tool-agnostic. vs plugins: MCP is open-source, not vendor-locked. vs custom integrations: MCP has auto-discovery and schema validation. MCP is becoming the USB-C of AI tool connectivity.
7. **Identify your MCP use cases** — map your workflow: "I frequently need to query a PostgreSQL database during development → PostgreSQL MCP server. I need to check GitHub issues → GitHub MCP server. I need live documentation → web search MCP server."
8. **Plan your MCP stack** — for a typical web development workflow: filesystem + GitHub + database + web search covers 90% of needs. For DevOps: add Docker + Kubernetes MCP servers. For data science: add Jupyter + pandas MCP servers.
9. **Understand transport mechanisms** — stdio transport: server runs as a subprocess, communicates via stdin/stdout. Best for local tools. SSE transport: server runs as an HTTP service, communicates via Server-Sent Events. Best for remote/shared tools.
10. **Follow MCP development** — the protocol is evolving rapidly (2024-2026). Track: `github.com/modelcontextprotocol/specification` for spec updates, Anthropic's blog for announcements, community servers on GitHub for new integrations.

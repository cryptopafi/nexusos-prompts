# VC-014 — Navigate GitHub Copilot Interface and Features

## Problema
Copilot's features are spread across inline completions, chat panel, inline chat, terminal integration, and code actions — developers typically discover only 2-3 of these, missing half the tool's capabilities.

## Procedura
### Prerequisites
- GitHub Copilot installed and configured (VC-013)

### Steps
1. **Map the five Copilot surfaces** — (a) Inline completions (ghost text in editor), (b) Chat panel (sidebar conversation), (c) Inline chat (Cmd+I, in-editor dialog), (d) Terminal suggestions (Cmd+I in terminal), (e) Code actions (lightbulb menu on errors/selections).
2. **Master inline completions** — these appear as gray ghost text while you type. `Tab` accepts, `Esc` dismisses, `Alt+]`/`Alt+[` cycles alternatives. Write a function name + comment, and Copilot generates the body. Effectiveness: highest for boilerplate and patterns.
3. **Master the Chat panel** — `Ctrl+Shift+I` opens the sidebar chat. Use for: explaining code, generating tests, asking architecture questions, debugging errors. Supports `@workspace`, `@file`, `@terminal` context. Maintains conversation history.
4. **Master Inline Chat** — `Cmd+I` with cursor in editor opens an inline dialog. Type: "Add error handling" or "Convert to async/await." AI edits directly in the editor with a diff preview. Accept/reject changes inline. Best for: targeted, in-place edits.
5. **Master Terminal Chat** — `Cmd+I` in the integrated terminal. Ask: "How do I find large files in this directory?" Copilot generates the command. Click to insert into terminal. Best for: shell commands you can't remember.
6. **Master Code Actions** — when Copilot detects an error (red squiggle) or you select code, a lightbulb appears. Click → "Fix with Copilot" or "Explain with Copilot." Also available via `Cmd+.` on selection. Best for: quick fixes and explanations.
7. **Use the Copilot status bar** — bottom-right shows Copilot status. Click for: enable/disable per file type, check usage quota, switch models, view suggestions log. Yellow = degraded performance. Red = disabled.
8. **Navigate Chat history** — Chat panel maintains history across sessions. Scroll up to reference previous conversations. Use "New Chat" (`+` button) to start fresh when context becomes stale (after ~20 messages).
9. **Use Chat participants** — type `@` in chat: `@workspace` = full project context, `@vscode` = VS Code API questions, `@terminal` = terminal output context. Each participant specializes Copilot's responses.
10. **Combine surfaces for complex tasks** — workflow: select code → `Cmd+I` inline chat "Explain this" → understand → Chat panel "Now refactor it using strategy pattern" → review diff → Code Action to fix any type errors.

### Verification
- [ ] Can access all 5 Copilot surfaces within 5 seconds each
- [ ] Inline chat edits code in-place with diff preview
- [ ] Terminal chat generates valid shell commands
- [ ] Code Actions appear on errors and selections
- [ ] Can combine 2+ surfaces in a single refactoring workflow

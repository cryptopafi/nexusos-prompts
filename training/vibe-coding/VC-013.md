# VC-013 — Set Up GitHub Copilot in VS Code

## Problema
Developers install Copilot but skip critical configuration steps — leaving performance settings at defaults, missing the Chat extension, and not setting up multi-model access — getting only 50% of Copilot's capability.

## Procedura
### Prerequisites
- VS Code 1.90+ installed
- GitHub account with Copilot subscription (Individual $10/mo, Business $19/mo, Enterprise $39/mo)
- Active internet connection

### Steps
1. **Install the Copilot extensions** — VS Code Extensions panel (`Cmd+Shift+X`) → install both: "GitHub Copilot" (inline completions) AND "GitHub Copilot Chat" (chat panel). Both are required for full functionality. Verify both show as enabled.
2. **Sign in to GitHub** — VS Code prompts for GitHub auth on first Copilot use. Click "Sign in to GitHub" → authorize in browser → return to VS Code. Verify: status bar shows Copilot icon (not crossed out).
3. **Configure the AI model** — Copilot now supports multiple models. Settings → search "copilot model" → select: Claude Sonnet 4 (balanced), GPT-4.1 (fast), or Claude Opus 4 (complex reasoning). Different models for different tasks.
4. **Enable Copilot Chat features** — Settings → search "copilot chat" → enable: `Inline Chat` (Cmd+I in editor), `Terminal Chat`, `Notebook Chat`. Enable `Code Actions` for AI-suggested fixes on errors.
5. **Configure inline completion behavior** — Settings → search "copilot inline" → set: `Enable Auto Completions: true`, `Suggestion Count: 3` (shows multiple options with Alt+]), `Delay: 50ms` (lower = more responsive, higher = fewer distractions).
6. **Set up language-specific settings** — enable/disable Copilot per language. Settings → search "copilot enable" → set per-language: enable for TypeScript, Python, Go; consider disabling for Markdown, JSON (reduces noise).
7. **Configure keyboard shortcuts** — essential bindings: `Tab` = accept suggestion, `Esc` = dismiss, `Alt+]` / `Alt+[` = cycle suggestions, `Cmd+I` = inline chat, `Ctrl+Shift+I` = Copilot Chat panel. Customize in Keyboard Shortcuts.
8. **Install complementary extensions** — `GitHub Pull Requests and Issues` (for PR integration), `Error Lens` (inline errors for Copilot to see), `GitLens` (git context). These provide context that improves Copilot's suggestions.
9. **Test inline completions** — create a new TypeScript file, type a function signature: `function calculateTax(income: number, rate: number):` and wait. Copilot should suggest the full function body. Accept with Tab. Test with 3 different functions.
10. **Test Chat** — open Copilot Chat (`Ctrl+Shift+I`), ask: "Explain the project structure" with `@workspace`. Then: "Write a unit test for @filename.ts". Verify: responses are contextual and accurate.

### Verification
- [ ] Both Copilot extensions installed and active (check Extensions panel)
- [ ] Status bar shows Copilot icon (enabled, not crossed out)
- [ ] Inline completions appear within 1 second of typing
- [ ] Chat responds to prompts with project-aware context
- [ ] Model selection is configured (not using default)

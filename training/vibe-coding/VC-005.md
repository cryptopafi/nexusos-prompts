# VC-005 — Debug Code Using AI-Assisted Iteration in Cursor

## Problema
Developers spend 40-60% of coding time debugging. Without a structured AI debugging workflow, they paste errors randomly into chat, miss root causes, and chase symptoms for hours.

## Procedura
### Prerequisites
- Cursor AI with Chat Panel access (VC-001)
- A project with a reproducible bug or error

### Steps
1. **Reproduce the error deterministically** — before involving AI, ensure the bug is reproducible. Note: exact error message, stack trace, and the action that triggers it. Copy the FULL error output — truncated errors lead to wrong AI diagnoses.
2. **Isolate the error scope** — determine: is it a compile error, runtime error, or logic error? Compile errors → show the file with the error. Runtime errors → include the stack trace + relevant source files. Logic errors → describe expected vs actual behavior with examples.
3. **Attach relevant context** — in Chat (`Cmd+L`), use `@filename` to attach the file causing the error PLUS its dependencies. For import errors, attach both the importing and imported files. Limit to 3-5 files maximum.
4. **Write a structured debug prompt** — format: "I'm getting this error: ```[full error]```. The code is in @file.ts, which imports from @utils.ts. Expected behavior: [X]. Actual behavior: [Y]. What's the root cause and fix?" Never just paste an error without context.
5. **Evaluate the AI's diagnosis** — AI will propose a root cause and fix. Before applying, verify: does the explanation match the error location? Does the fix address the root cause or just suppress the symptom? Ask "Why does this fix work?" to validate understanding.
6. **Apply and test incrementally** — click "Apply" on the suggested fix. Run the code immediately. If the error changes (different error), share the new error: "That fixed the import error but now I get: [new error]." Iterate until clean.
7. **Use breakpoint-assisted debugging** — for logic errors, ask: "Where should I place breakpoints to trace the flow of [variable/data] through this function?" Set breakpoints, run debugger (`F5`), and share the variable values with AI for analysis.
8. **Handle multi-layer bugs** — for errors spanning frontend-backend or multiple services, debug layer by layer. Start with the layer closest to the error (usually the stack trace top), fix it, then move to the next layer.
9. **Document the fix** — once resolved, add a comment explaining the bug and fix. For recurring patterns, create a `KNOWN_ISSUES.md`. This prevents re-encountering the same bug.
10. **Build a debug prompt library** — save effective debug prompts in your `prompts/debug/` directory. Categories: type errors, async/await issues, import resolution, API errors, database connection errors. Reuse patterns across projects.

### Verification
- [ ] Error is fully resolved (no error output, expected behavior confirmed)
- [ ] Root cause is understood (can explain why the bug occurred)
- [ ] Fix doesn't introduce new bugs (run existing tests)
- [ ] Debug resolution completed in <3 AI iterations for common errors
- [ ] Fix documented with inline comment

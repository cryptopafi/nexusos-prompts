# VC-001 — Set Up Cursor AI IDE for Vibe Coding

## Problema
New vibe coders lose 30-60 minutes wrestling with installation, model selection, and extension migration — delaying their first AI-assisted coding session unnecessarily.

## Procedura
### Prerequisites
- macOS 12+ / Windows 10+ / Ubuntu 20.04+
- 8GB+ RAM (16GB recommended for local model inference)
- Active internet connection for cloud model access

### Steps
1. **Download Cursor AI** from `cursor.com` — select the correct OS build (macOS ARM for M-series, x64 for Intel). Verify checksum if provided. Current stable: v0.45+.
2. **Run the installer** and grant necessary permissions: macOS → Full Disk Access + Accessibility; Windows → allow firewall for `cursor.exe`; Linux → `chmod +x` the AppImage or install via `.deb`.
3. **Import VS Code settings** — on first launch, Cursor offers a one-click migration (extensions, keybindings, settings.json). Accept to preserve muscle memory. Benchmark: migration of 40 extensions takes <2 minutes.
4. **Sign in or create a Cursor account** — free tier includes 2,000 completions/month + 50 slow premium requests. Pro ($20/mo) adds 500 fast premium requests. Select based on usage volume.
5. **Select your AI model** — Settings → Models. Recommended: Claude Sonnet 4.6 for speed, Claude Opus 4 for complex reasoning, GPT-4.1 as fallback. Enable "long context mode" for files >500 lines.
6. **Configure essential settings** — enable `Cursor Tab` (predictive completions), set `Apply Model` to same as chat model, enable `Auto-import` for TypeScript/Python, set `Max Context Lines` to 200+.
7. **Install critical extensions**: `ESLint`, `Prettier`, `GitLens`, `Error Lens`, `TODO Highlight`. For Python: `Pylance` + `Black Formatter`. For web: `Tailwind CSS IntelliSense`.
8. **Create a `.cursorrules` file** in your home directory with global preferences (language, style, framework defaults). Example: `Always use TypeScript strict mode. Prefer functional components. Use Tailwind for styling.`
9. **Test the setup** — create a new project folder, open it in Cursor, press `Cmd+L` to open chat, type "Create a Hello World Express server with TypeScript" and verify code generation works end-to-end (generation → file creation → terminal run).
10. **Benchmark your workflow** — time the full cycle: prompt → generated code → running app. Target: <60 seconds for a simple endpoint. If slower, check model selection or network latency.

### Verification
- [ ] Cursor opens without errors, shows correct version in Help → About
- [ ] AI chat responds to prompts within 3 seconds (fast model)
- [ ] VS Code extensions migrated and functional (verify ESLint shows warnings)
- [ ] `.cursorrules` file loads (check via Cmd+Shift+P → "Cursor: Show Rules")
- [ ] Test project compiles and runs successfully from generated code

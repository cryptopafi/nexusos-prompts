# VC-007 — Choose Between Ask Mode vs Agent Mode in Cursor

## Problema
Developers default to one mode for everything — either over-using Agent mode (burning tokens on simple questions) or under-using it (manually applying multi-file changes from Chat), losing 20-40% efficiency.

## Procedura
### Prerequisites
- Cursor AI with both modes available (VC-001)
- Understanding of Chat Panel and Composer (VC-002)

### Steps
1. **Understand the mode spectrum** — Ask/Chat (`Cmd+L`) = read-only AI assistant (answers questions, generates snippets). Composer (`Cmd+I`) = AI editor (applies changes to files). Agent (toggle in Composer) = autonomous AI developer (creates files, runs commands, iterates).
2. **Use Ask Mode for**: questions about code ("What does this function do?"), explaining errors, architecture advice, one-off code snippets (<20 lines), learning new APIs. Cost: low token usage. Speed: instant.
3. **Use Composer (non-Agent) for**: targeted edits to 1-2 files, refactoring a specific function, adding a method to an existing class, updating imports. The AI generates a diff you review before applying. Cost: moderate tokens. Speed: fast.
4. **Use Agent Mode for**: scaffolding new projects, features spanning 3+ files, tasks requiring terminal commands (install packages, run migrations), fixing build errors that need iterative debugging. Cost: high tokens. Speed: depends on iterations.
5. **Decision framework** — answer these questions: (a) Do I need files created/deleted? → Agent. (b) Do I need terminal commands? → Agent. (c) Am I editing 1-2 existing files? → Composer. (d) Do I just need information? → Ask. (e) Am I unsure what to do? → Ask first, then escalate.
6. **Avoid Agent Mode anti-patterns** — don't use Agent for: simple typo fixes, formatting changes, questions about code, reading documentation. Agent mode burns 5-10x more tokens than Ask mode for these tasks.
7. **Chain modes effectively** — start with Ask: "How should I structure a caching layer for this app?" Read the plan. Then switch to Agent: "Implement the caching layer as described above using Redis + ioredis." Ask → plan, Agent → execute.
8. **Monitor token usage** — check Cursor's usage dashboard (Settings → Usage). Track tokens per mode per week. Target: 60% Ask, 25% Composer, 15% Agent. If Agent usage exceeds 30%, you're likely over-using it.
9. **Set per-mode model preferences** — use a faster/cheaper model for Ask (Sonnet), and a more capable model for Agent (Opus or GPT-4.1). Configure in Settings → Models per feature.
10. **Build mode-switching reflexes** — practice switching: `Cmd+L` for Ask, `Cmd+I` for Composer, toggle for Agent. After 1 week, mode selection should be automatic based on task type.

### Verification
- [ ] Can articulate when to use each mode with 3 examples
- [ ] Token usage drops 20%+ after optimizing mode selection
- [ ] Agent mode used only for multi-file or terminal-requiring tasks
- [ ] Mode switching takes <2 seconds via keyboard shortcuts

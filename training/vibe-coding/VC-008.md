# VC-008 — Use Context Tools (@file, @folder, @web) in Cursor

## Problema
Developers paste code manually into chat instead of using @-mentions, losing file references, line numbers, and project structure — resulting in 40% less accurate AI responses.

## Procedura
### Prerequisites
- Cursor AI configured (VC-001)
- Project with multiple files open

### Steps
1. **Master @file references** — type `@` in Chat to see autocomplete. `@filename.ts` attaches the full file content as context. Use for: "Refactor @utils.ts to use async/await" or "The bug is in @api.controller.ts line 45."
2. **Use @folder for directory context** — `@src/routes` attaches all files in the routes directory. Use when the AI needs to understand a module's structure. Limit: Cursor truncates if total context exceeds the model's window.
3. **Use @codebase for project-wide questions** — `@codebase` indexes the entire project. Best for: "How does authentication work in this project?" or "Find all usages of the User model." Warning: slower and more expensive than targeted @file.
4. **Use @web for live documentation** — `@web` triggers a web search. Use for: "Using @web, find the latest Prisma migration syntax" or "What's the current API for NextAuth v5?" Keeps AI responses up-to-date with latest library versions.
5. **Use @doc for documentation context** — `@doc` references Cursor's built-in documentation index. Use for framework-specific questions: "@doc NextJS app router — how do I create a dynamic route?"
6. **Combine context tools strategically** — example: "Looking at @models/user.ts and @routes/auth.ts, and checking @web for the latest bcrypt best practices, add password hashing to the registration endpoint." Multiple @-mentions compound AI accuracy.
7. **Use selection + @file together** — highlight a code block, press `Cmd+L`, then add `@related-file.ts`. The AI sees both the selected code AND the referenced file. Best for: cross-file refactoring, fixing imports.
8. **Control context window budget** — each @-mention consumes tokens. Budget: 2-3 @file for focused tasks, 1 @folder for module understanding, @codebase only for exploration. Never combine @codebase + @web (too expensive and unfocused).
9. **Use @git for change context** — `@git` shows recent diffs. Use: "Looking at @git, summarize what changed in the last 3 commits" or "Given @git, did I introduce any bugs in today's changes?"
10. **Build context habits** — for every prompt, ask: "What context does the AI need to answer correctly?" Add exactly that via @-mentions. Prompt without context = generic answer. Prompt with targeted context = accurate, project-specific answer.

### Verification
- [ ] @file correctly attaches file content (visible in chat header)
- [ ] @folder shows all files in the target directory
- [ ] @web returns current documentation (test with a recent API change)
- [ ] Combined @-mentions produce more accurate responses than bare prompts
- [ ] Context budget stays within model limits (no truncation warnings)

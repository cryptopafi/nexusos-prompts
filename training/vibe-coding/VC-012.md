# VC-012 — Build and Integrate an MCP Server with Cursor

## Problema
Developers rely solely on pre-built MCP servers, unable to create custom servers for their specific tools and APIs — missing 60% of MCP's value for unique workflows.

## Procedura
### Prerequisites
- Understanding of MCP concepts (VC-011)
- Node.js 18+ or Python 3.10+
- Cursor AI installed (VC-001)

### Steps
1. **Choose your MCP server framework** — TypeScript: `@modelcontextprotocol/sdk` (official, best maintained). Python: `mcp` package (pip install mcp). Both provide server scaffolding, tool registration, and transport handling.
2. **Scaffold the server** — TypeScript: `npm init -y && npm install @modelcontextprotocol/sdk zod`. Create `src/index.ts`. Python: `pip install mcp` → create `server.py`. Define the server name and version.
3. **Define your tools** — each tool needs: name, description, input schema (JSON Schema or Zod), handler function. Example: a "query_notion" tool that takes a database_id and filter, calls the Notion API, and returns results.
4. **Implement tool handlers** — write the actual logic. Each handler receives validated input and returns a result. Handle errors gracefully — return error messages, don't throw. Include rate limiting if calling external APIs.
5. **Add input validation** — use Zod (TypeScript) or Pydantic (Python) for strict input validation. Define schemas for every tool parameter. This prevents AI from sending malformed requests.
6. **Implement the transport layer** — for Cursor, use stdio transport (server runs as subprocess). TypeScript: `server.connect(new StdioServerTransport())`. Python: `mcp.server.stdio.run_server()`. Cursor launches the server process automatically.
7. **Configure Cursor to use your server** — add to `.cursor/mcp.json` in your project or global Cursor settings: `{"mcpServers": {"my-server": {"command": "node", "args": ["path/to/dist/index.js"]}}}`. Restart Cursor.
8. **Test the integration** — in Cursor Chat, the AI should now list your tools when relevant. Prompt: "Use the query_notion tool to find all tasks with status 'In Progress'." Verify the tool is called and results are returned.
9. **Add error handling and logging** — log all tool calls to a file for debugging: timestamp, tool name, input, output, duration. Handle API timeouts, authentication errors, and rate limits with informative error messages.
10. **Package and distribute** — add a README, publish to npm/PyPI if reusable. For team use, add the MCP config to the project's `.cursor/mcp.json` and commit to git. Other team members get the server automatically.

### Verification
- [ ] MCP server starts without errors via stdio
- [ ] Cursor discovers and lists the custom tools
- [ ] AI successfully calls a custom tool and returns results
- [ ] Input validation rejects malformed requests
- [ ] Error handling returns informative messages (not crashes)

# VC-002 — Navigate and Configure Cursor AI Interface

## Problema
Users unfamiliar with Cursor AI's layout waste 15-20 minutes per session hunting for panels, missing powerful features hidden behind keyboard shortcuts.

## Procedura
### Prerequisites
- Cursor AI installed and running (VC-001)

### Steps
1. **Learn the core layout** — Editor (center), File Explorer (left sidebar), Chat Panel (right sidebar), Terminal (bottom). Press `Cmd+B` to toggle sidebar, `Cmd+J` to toggle terminal.
2. **Master the Chat Panel** — `Cmd+L` opens inline chat. This is your primary AI interface. Type natural language prompts. Use `@` mentions to add context. Chat history persists per session — use `Cmd+N` in chat for a fresh conversation.
3. **Master the Composer/Agent Panel** — `Cmd+I` opens Composer for multi-file edits. Agent mode (toggle in Composer) can create files, run terminal commands, and iterate autonomously. Use Composer for tasks spanning 3+ files.
4. **Configure the Activity Bar** — right-click the left sidebar to show/hide: Source Control, Extensions, Search, Debug. Pin the ones you use daily. Hide Accounts and Extensions if rarely used.
5. **Set up keyboard shortcuts** — `Cmd+Shift+P` → "Keyboard Shortcuts". Essential custom bindings: `Cmd+Shift+L` for "Add to Chat", `Cmd+K` for inline edit, `Alt+\` for Cursor Tab accept. Export keybindings.json for backup.
6. **Configure the terminal** — Settings → Terminal → set default shell (zsh recommended). Enable `terminal.integrated.copyOnSelection`. Set font to a Nerd Font (e.g., `JetBrainsMono Nerd Font`) for icon support.
7. **Set up split editors** — `Cmd+\` splits vertically. Drag tabs between splits. Use this for side-by-side AI output vs source file comparison. `Cmd+1/2/3` switches between editor groups.
8. **Configure theme and readability** — Settings → Color Theme → recommended: `One Dark Pro` or `GitHub Dark`. Set font size 14-16px, line height 1.6, enable `Editor: Minimap` for large files, bracket pair colorization ON.
9. **Pin and arrange frequently used panels** — drag Chat panel to bottom or secondary sidebar. Right-click tabs → "Keep Open" to prevent auto-close. Use `Cmd+Shift+E` for Explorer, `Cmd+Shift+G` for Git, `Cmd+Shift+F` for Search.
10. **Create a workspace file** — File → Save Workspace As → `.code-workspace`. Include folder paths, workspace-specific settings (model selection, rules path). Share with team for consistent setup.

### Verification
- [ ] Can open Chat, Composer, Terminal, Explorer each in <2 seconds via shortcuts
- [ ] Custom keybindings saved and functional (test 3 custom bindings)
- [ ] Split editor layout works with AI output on one side, source on the other
- [ ] Workspace file saves and restores layout correctly on restart

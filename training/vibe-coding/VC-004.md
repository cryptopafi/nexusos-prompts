# VC-004 — Build a Full App with Cursor AI Agent Mode

## Problema
Developers attempt to build entire applications through manual Chat prompts, missing Agent Mode's ability to autonomously create files, install packages, run commands, and iterate on errors — resulting in 3-5x slower development.

## Procedura
### Prerequisites
- Cursor AI Pro plan (Agent Mode requires premium model access)
- Node.js 18+ or Python 3.10+ installed
- Empty project directory ready

### Steps
1. **Open Composer in Agent Mode** — `Cmd+I` to open Composer, then toggle "Agent" mode (switch at top of panel). Agent mode can: create/edit files, run terminal commands, read error output, and self-correct. Standard Composer only edits.
2. **Write a comprehensive app specification** — describe the full app in one prompt: tech stack, features, file structure, UI requirements. Example: "Build a task management API with Express + TypeScript + Prisma + SQLite. Include: CRUD endpoints for tasks, user authentication with JWT, input validation with Zod, error middleware. Use a clean folder structure with routes/, controllers/, middleware/, prisma/."
3. **Let Agent scaffold the project** — Agent will create `package.json`, install dependencies, set up `tsconfig.json`, create folder structure, and write initial files. Watch the terminal — Agent runs `npm install`, `npx prisma init`, etc. autonomously.
4. **Review the generated structure** — check that file organization matches your spec. Common Agent patterns: `src/index.ts` (entry), `src/routes/` (endpoints), `src/middleware/` (auth, error handling), `prisma/schema.prisma` (data model). Flag any structural issues immediately.
5. **Iterate feature-by-feature** — after scaffolding, add features incrementally: "Now add rate limiting with express-rate-limit, 100 requests per 15 minutes per IP." Agent will install the package, create middleware, and wire it into the app.
6. **Let Agent debug autonomously** — when Agent runs code and hits errors, it reads the error output, identifies the fix, and applies it. Allow 3-5 self-correction cycles before intervening. Success rate for common errors (missing imports, type mismatches): ~85%.
7. **Add tests via Agent** — prompt: "Write Jest tests for all task CRUD endpoints. Include setup/teardown with test database. Test both success and error cases." Agent creates test files, installs Jest + supertest, configures `jest.config.ts`.
8. **Run the full application** — instruct Agent: "Start the dev server and run all tests." Verify: server starts on expected port, all tests pass, API responds correctly to curl/Postman requests.
9. **Refine with constraints** — add non-functional requirements: "Add request logging with Winston, structured JSON format. Add Swagger docs at /api-docs using swagger-jsdoc." Agent handles multi-file changes coherently.
10. **Export and document** — ask Agent: "Create a README.md with setup instructions, API documentation, and environment variable list." Review for accuracy. Commit the full project.

### Verification
- [ ] Agent creates 10+ files autonomously in correct structure
- [ ] Application starts without errors after Agent scaffolding
- [ ] At least 5 tests pass on first run
- [ ] Agent self-corrects at least 2 errors without human intervention
- [ ] Full app (CRUD + auth + validation + tests) built in <30 minutes

# VC-003 — Use Cursor AI Chat Panel for Code Generation

## Problema
Developers underutilize the Chat Panel by writing vague prompts, getting generic code that requires heavy manual editing — negating the speed advantage of AI-assisted coding.

## Procedura
### Prerequisites
- Cursor AI configured with preferred model (VC-001, VC-002)
- A project open with at least one source file for context

### Steps
1. **Open Chat Panel** — `Cmd+L`. Select your model (Claude Sonnet 4.6 for speed, Opus 4 for complex logic). The model selector is at the top of the chat panel.
2. **Write structured prompts** — follow the pattern: `[ROLE] + [CONTEXT] + [TASK] + [CONSTRAINTS]`. Example: "You are a TypeScript expert. Given the Express app in @server.ts, add a /users endpoint that returns paginated JSON. Use Zod for validation, limit 50 items per page."
3. **Add file context with @mentions** — use `@filename` to attach specific files, `@folder` for directories, `@codebase` for full project context. Rule: include only relevant files — more context ≠ better results. 2-3 targeted files > entire codebase.
4. **Use selection-based prompting** — highlight code in the editor, press `Cmd+L`, and the selection auto-attaches to chat. Ask: "Refactor this function to use async/await" or "Add error handling to this block."
5. **Iterate on responses** — if the first response is 80% correct, don't start over. Say: "Good, but change X to Y and add Z." Cursor maintains conversation context. Average iterations to production code: 2-3.
6. **Apply generated code** — click "Apply" on code blocks in chat. Cursor diffs against your current file and merges intelligently. Review the diff before accepting (green = additions, red = deletions).
7. **Use chat for debugging** — paste error messages directly: "I'm getting this error: [paste]. The relevant code is in @file.ts. Fix it." Include the full stack trace — AI models parse stack traces effectively.
8. **Chain prompts for complex features** — break large features into 3-5 sequential prompts. Prompt 1: data model. Prompt 2: API endpoint. Prompt 3: validation. Prompt 4: tests. Each builds on the previous.
9. **Save useful prompts as snippets** — create a `prompts/` directory in your project with `.md` files of proven prompts. Reuse across projects. Effective prompts are an asset.
10. **Measure generation quality** — track: (a) lines generated vs lines manually edited, (b) time-to-working-code, (c) bugs introduced by AI. Target: >70% of generated code accepted without edits.

### Verification
- [ ] Can generate a functional API endpoint from a single well-structured prompt
- [ ] @mentions correctly attach file context (verify in chat header)
- [ ] Apply button merges code without breaking existing functionality
- [ ] Debug workflow resolves an error in <3 prompt iterations

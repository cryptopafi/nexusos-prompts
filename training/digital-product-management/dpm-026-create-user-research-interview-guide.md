# DPM-026 — Create User Research Interview Guide

## Problema
Interviews are unstructured, leading to biased data and missed insights.

## Procedura
### Prerequisites
- Research objective defined
- Target user segment identified

### Steps
1. Define the research objective: 'We want to learn [specific thing] about [user segment]'
2. Structure the guide: warm-up (5 min), core questions (25 min), wrap-up (5 min)
3. Write 3 warm-up questions to build rapport (e.g., 'Tell me about your role')
4. Write 8-10 core questions: open-ended, non-leading, behavior-focused
5. Add probing follow-ups for each core question ('Tell me more', 'Walk me through that', 'Why?')
6. Include one 'magic wand' question: 'If you could change one thing about [X], what would it be?'
7. Pilot the guide with one participant, adjust timing and question clarity

### Verification
- Interview guide document created
- Guide piloted and refined
- Ready for recruitment

# DPM-075 — Plan and Run Sprint Iteration

## Problema
Sprints are poorly planned, leading to unfinished work, scope creep, and demoralized teams.

## Procedura
### Prerequisites
- Estimated backlog (DPM-073)
- Team velocity data
- Sprint cadence (usually 2 weeks)

### Steps
1. Calculate team velocity: average story points completed in last 3 sprints
2. Sprint Planning: select stories totaling velocity from the top of the prioritized backlog
3. Define the sprint goal: a one-sentence description of what this sprint achieves for users
4. Break stories into tasks and assign owners (during planning or within first day)
5. Track progress daily: use burndown chart or Kanban board
6. Manage scope: if new work appears mid-sprint, it trades with existing work or waits for next sprint
7. Sprint close: review delivered work, run retrospective, update velocity

### Verification
- Sprint planned with goal and velocity-based scope
- Daily tracking active
- Sprint completed with retro

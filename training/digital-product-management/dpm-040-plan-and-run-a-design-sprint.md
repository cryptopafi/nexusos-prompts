# DPM-040 — Plan and Run a Design Sprint

## Problema
Teams spend weeks debating solutions when a structured 5-day process could produce and test a prototype.

## Procedura
### Prerequisites
- Cross-functional team (PM, design, engineering, stakeholder)
- 5 consecutive days blocked
- User recruits for day 5

### Steps
1. Day 1 (Understand): map the challenge, set a sprint goal, interview experts, pick a target
2. Day 2 (Diverge): generate solutions individually using Crazy 8s and solution sketches
3. Day 3 (Decide): present solutions, vote using dot voting, create a storyboard for the winner
4. Day 4 (Prototype): build a realistic-looking prototype (not functional, just enough to test)
5. Day 5 (Test): run 5 user tests against the prototype, observe and take notes
6. Debrief: synthesize test results, identify patterns, make go/no-go/iterate decision
7. Document sprint outcomes: decision, next steps, key learnings

### Verification
- 5-day sprint completed
- Prototype tested with 5 users
- Decision and learnings documented

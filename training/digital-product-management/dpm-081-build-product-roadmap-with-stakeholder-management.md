# DPM-081 — Build Product Roadmap with Stakeholder Management

## Problema
Roadmaps fail because they are built in isolation and don't account for stakeholder expectations and constraints.

## Procedura
### Prerequisites
- Product strategy
- Stakeholder input
- Prioritized backlog

### Steps
1. Gather stakeholder inputs: run 1:1 meetings with key stakeholders to understand their priorities and constraints
2. Synthesize inputs: find common themes and conflicts across stakeholders
3. Build roadmap draft using Now/Next/Later format (avoid dates where possible, use outcomes)
4. For each roadmap item, link to: user problem, business metric, and stakeholder sponsor
5. Present the roadmap to stakeholders: explain what's included, what's not, and why
6. Handle pushback: use prioritization framework data to justify trade-offs
7. Set a quarterly roadmap review cadence with stakeholders to maintain alignment

### Verification
- Stakeholder inputs gathered
- Roadmap built with linked metrics
- Quarterly review cadence set

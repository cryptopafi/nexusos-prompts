# DPM-020 — Build Product Prioritization Framework and Roadmap

## Problema
Features are prioritized by loudest voice instead of data, leading to scattered product direction.

## Procedura
### Prerequisites
- Feature backlog
- Business goals
- User research data

### Steps
1. Choose a prioritization framework: RICE (Reach, Impact, Confidence, Effort), ICE, or Value vs. Effort
2. Score all backlog items using the chosen framework
3. Group scored items into themes aligned with product strategy
4. Define roadmap time horizons: Now (current quarter), Next (next quarter), Later (future)
5. Place top-scored items in 'Now', create a balanced mix across themes
6. Build a visual roadmap using a tool (Productboard, Aha!, or simple spreadsheet)
7. Present roadmap to stakeholders, explain prioritization rationale, and collect feedback

### Verification
- All backlog items scored
- Roadmap built with Now/Next/Later
- Roadmap presented to stakeholders

# DPM-049 — Prioritize User Outcomes with Story Mapping

## Problema
Backlogs are flat lists that lack context about user journey and priority trade-offs.

## Procedura
### Prerequisites
- User stories or epics
- User journey understanding
- Team collaboration

### Steps
1. Define the user journey backbone: the high-level steps users take (left to right)
2. Under each backbone step, list all user activities as cards
3. Arrange activities vertically by priority: must-have on top, nice-to-have below
4. Draw a horizontal line across the map: everything above = MVP, below = later releases
5. Identify gaps: are there backbone steps with no activities? Those need discovery
6. Use the story map in sprint planning to select a coherent slice of value
7. Update the story map weekly as new stories are added or priorities change

### Verification
- Story map created with backbone and activities
- MVP line drawn
- Used in sprint planning

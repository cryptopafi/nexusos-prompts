# DPM-047 — Write Usability Test Plan

## Problema
Usability tests are run ad-hoc without a plan, producing unreliable and non-actionable results.

## Procedura
### Prerequisites
- Feature or prototype to test
- Usability hypothesis (DPM-037)
- Access to users

### Steps
1. Define the test objective: what usability questions must be answered?
2. Select test method: moderated (live) or unmoderated (recorded), remote or in-person
3. Define participant criteria: 5 users matching your target persona
4. Write 3-5 task scenarios in user language (not system language): 'You want to [goal]. Please show me how.'
5. Define metrics per task: completion rate, time on task, error count, satisfaction rating (SUS or single-ease)
6. Prepare the test script: introduction, consent, warm-up, tasks, debrief
7. Prepare the observation template: columns for task, success, time, errors, quotes

### Verification
- Test plan document completed
- Task scenarios and metrics defined
- Script and observation template ready

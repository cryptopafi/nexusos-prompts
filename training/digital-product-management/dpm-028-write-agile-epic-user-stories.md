# DPM-028 — Write Agile Epic User Stories

## Problema
Epics are written as vague feature descriptions instead of structured stories that guide development.

## Procedura
### Prerequisites
- Product backlog
- Persona or JTBD data
- Sprint planning cadence

### Steps
1. Define what makes a good epic: large user story that spans multiple sprints
2. Write the epic in format: 'As a [persona], I want to [capability], so that [outcome]'
3. Add business context: why this epic matters now, what metric it impacts
4. Define acceptance criteria at the epic level: what must be true when the epic is complete?
5. Estimate epic size: T-shirt sizing (S, M, L, XL) based on complexity and uncertainty
6. Identify dependencies: what must exist before this epic can start?
7. Add the epic to the roadmap with a target quarter

### Verification
- Epic written in user story format
- Acceptance criteria defined
- Epic sized and roadmapped

# DPM-030 — Storyboard an Epic

## Problema
Teams jump into building without visualizing the user flow, causing UX inconsistencies and missing steps.

## Procedura
### Prerequisites
- Epic user story
- User flow knowledge
- Whiteboard or digital tool

### Steps
1. Identify the starting point: what triggers the user to begin this epic's flow?
2. Sketch each step as a panel: what the user sees, does, and feels
3. Include decision points: where does the user choose between paths?
4. Show error states and edge cases as alternate panels
5. Add annotations: technical requirements, API calls, data needed at each step
6. Review the storyboard with design and engineering for feasibility
7. Use the storyboard to derive child stories for sprint planning

### Verification
- Storyboard created with all steps and decision points
- Reviewed with team
- Child stories derived

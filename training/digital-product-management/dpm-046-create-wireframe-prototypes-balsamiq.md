# DPM-046 — Create Wireframe Prototypes (Balsamiq)

## Problema
Product ideas are communicated verbally, causing misalignment between PM, design, and engineering.

## Procedura
### Prerequisites
- Balsamiq or similar wireframe tool
- User stories with acceptance criteria
- Basic UX knowledge

### Steps
1. Identify the user flow to wireframe (tied to a specific user story or epic)
2. Start with the entry point: what is the first screen the user sees?
3. Wireframe each step in the flow: keep it low-fidelity (boxes, lines, text only)
4. Add interactive hotspots to link screens into a clickable prototype
5. Include annotations explaining behavior that isn't visible (e.g., 'dropdown loads from API')
6. Review the prototype with engineering to confirm technical feasibility
7. Use the prototype in a usability test or stakeholder review before moving to high-fidelity design

### Verification
- Wireframe prototype created with linked screens
- Engineering review completed
- Prototype used for feedback

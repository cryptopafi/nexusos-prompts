# DPM-062 — Manage Data Science/AI Team as Product Manager

## Problema
PMs struggle to manage data science teams because DS workflow (experimentation, iteration) differs from traditional engineering.

## Procedura
### Prerequisites
- Data science team access
- Understanding of ML/AI workflow
- Product goals

### Steps
1. Understand the DS workflow: problem framing → data collection → exploration → modeling → evaluation → deployment
2. Define the product problem clearly before DS starts (avoid 'find something interesting in the data')
3. Set success metrics that DS can optimize against (accuracy, precision, recall, business KPI)
4. Create a shared backlog with DS: experiments, data tasks, model improvements, deployment tasks
5. Establish a weekly sync: DS presents findings, PM connects to product decisions
6. Plan for iteration: DS rarely gets it right the first time, budget 2-3 iterations per project
7. Bridge the communication gap: translate DS findings into product and business language for stakeholders

### Verification
- DS backlog created
- Success metrics defined
- Weekly sync established with stakeholder reporting

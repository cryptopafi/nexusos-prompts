# DPM-068 — Run XP Practices

## Problema
Engineering quality suffers because XP practices like pair programming and TDD are not adopted.

## Procedura
### Prerequisites
- Engineering team
- XP knowledge
- Development environment

### Steps
1. Introduce pair programming: developers work in pairs on complex or critical code
2. Implement Test-Driven Development (TDD): write test first, then code to pass the test
3. Set up continuous integration: code is integrated and tested multiple times per day
4. Practice collective code ownership: any developer can modify any code (no silos)
5. Implement coding standards: team agrees on style, naming, and patterns
6. Run weekly refactoring sessions: dedicate time to improve code quality
7. Measure impact: defect rate, deployment frequency, code review turnaround time

### Verification
- Pair programming sessions happening
- TDD practiced on new features
- CI running with team standards

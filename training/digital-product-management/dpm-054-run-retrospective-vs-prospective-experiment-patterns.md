# DPM-054 — Run Retrospective vs Prospective Experiment Patterns

## Problema
Teams only run forward-looking experiments, missing the insights available from historical data analysis.

## Procedura
### Prerequisites
- Historical product data
- Analytics tool
- Experiment design knowledge

### Steps
1. Understand the difference: retrospective = analyze past data; prospective = design and run new test
2. Identify questions that can be answered retrospectively: 'Did feature X improve retention for users who adopted it?'
3. Run a retrospective analysis: pull historical cohorts, compare outcomes, control for confounders
4. Identify questions that require prospective experiments: 'Will changing Y improve conversion?'
5. Design and run a prospective experiment with proper control group
6. Compare the insights from retrospective and prospective approaches
7. Document when to use each pattern: retrospective for exploration, prospective for validation

### Verification
- Retrospective analysis completed
- Prospective experiment run
- Pattern guidance documented

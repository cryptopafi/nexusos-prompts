# DPM-011 — Design for User Habits (Hook Framework)

## Problema
Products fail to retain users because they don't create habitual engagement loops.

## Procedura
### Prerequisites
- Product with existing user flow
- Analytics data on user behavior

### Steps
1. Define the internal trigger: what emotion or situation prompts the user to open your product?
2. Design the external trigger: notification, email, or prompt that brings users back
3. Define the action: the simplest behavior the user takes in anticipation of a reward
4. Design the variable reward: what unpredictable value does the user get? (social, hunt, self)
5. Design the investment: what does the user put in that makes the product better for next use?
6. Map your current product to the Hook Model and identify the weakest link
7. Redesign the weakest link and test the improvement with a cohort of users

### Verification
- Hook Model mapped for product
- Weakest link identified
- Improvement designed and tested

# DPM-048 — Run a Usability Design Sprint

## Problema
Traditional usability improvement cycles take weeks when a focused sprint could fix critical issues in days.

## Procedura
### Prerequisites
- Usability test results
- Cross-functional team
- 3-5 days blocked

### Steps
1. Day 1: Review usability test results, prioritize top 3 usability failures by severity
2. Day 2: Brainstorm solutions for each failure, sketch 2-3 alternatives per problem
3. Day 2: Vote on best solutions, create a plan for the prototype
4. Day 3: Build an updated prototype incorporating the top solutions
5. Day 4: Run 5 usability tests on the new prototype with the same task scenarios
6. Day 5: Compare before/after metrics, document improvements and remaining issues
7. Deliver: improvement specs to engineering, updated design files, test results report

### Verification
- 3 usability failures addressed
- New prototype tested
- Before/after comparison documented

# DPM-031 — Draft Child Stories from Epics

## Problema
Epics are too large to execute in sprints, and teams struggle to break them into right-sized stories.

## Procedura
### Prerequisites
- Epic user story (DPM-028)
- Storyboard (DPM-030)
- Sprint capacity knowledge

### Steps
1. Review the epic and its storyboard to identify natural breakpoints
2. Apply INVEST criteria to each potential child story: Independent, Negotiable, Valuable, Estimable, Small, Testable
3. Write each child story in standard format with acceptance criteria
4. Size each story: should be completable within one sprint (typically 1-8 story points)
5. Order stories by dependency and value: which must come first? which delivers value soonest?
6. Create a story map: arrange stories by user journey (horizontal) and priority (vertical)
7. Review the story set with the team: are they clear, sized right, and properly ordered?

### Verification
- Child stories written with acceptance criteria
- Stories sized and ordered
- Story map created

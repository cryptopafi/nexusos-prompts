# DPM-082 — Scale Agile (Spotify Model, SAFe Intro)

## Problema
Single-team agile practices break down when multiple teams must coordinate on a shared product.

## Procedura
### Prerequisites
- Multiple agile teams
- Scaling framework knowledge
- Organizational leadership support

### Steps
1. Assess scaling readiness: do individual teams have mature agile practices? (if not, fix that first)
2. Study scaling frameworks: Spotify Model (squads/tribes/chapters/guilds), SAFe (ARTs/PIs), LeSS
3. Choose the framework that fits your organization's culture and size
4. For Spotify: organize squads around product areas, create chapters for functional expertise
5. For SAFe: set up Agile Release Trains, plan Program Increments (PIs) quarterly
6. Implement inter-team coordination: Scrum of Scrums, shared backlog reviews, dependency mapping
7. Measure scaling effectiveness: delivery predictability, cross-team dependency resolution time, team satisfaction

### Verification
- Scaling framework selected with rationale
- Initial structure implemented
- Scaling metrics tracked

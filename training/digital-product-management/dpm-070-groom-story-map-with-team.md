# DPM-070 — Groom Story Map with Team

## Problema
Story maps become outdated because they aren't regularly groomed with the team.

## Procedura
### Prerequisites
- Existing story map (DPM-049)
- Cross-functional team
- 1-hour meeting slot

### Steps
1. Schedule a bi-weekly story map grooming session (1 hour)
2. Review the backbone: is the user journey still accurate? Add or remove steps as needed
3. Review activities under each step: are priorities still correct? Reprioritize if needed
4. Add new stories that have emerged from user research, bugs, or stakeholder requests
5. Remove or archive completed stories
6. Adjust the MVP line if scope or strategy has changed
7. Use the groomed map to feed the next sprint planning session

### Verification
- Story map reviewed and updated
- New stories added, completed stories archived
- Map used for sprint planning

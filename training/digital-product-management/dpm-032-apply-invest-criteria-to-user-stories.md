# DPM-032 — Apply INVEST Criteria to User Stories

## Problema
User stories are poorly written, causing confusion during sprint planning and development.

## Procedura
### Prerequisites
- Backlog of user stories
- INVEST framework knowledge

### Steps
1. Review each story against Independent: can it be developed without depending on other stories?
2. Check Negotiable: is it flexible enough to discuss implementation with the team?
3. Check Valuable: does it deliver value to the user or business?
4. Check Estimable: can the team estimate effort? If not, it needs a spike or more discovery
5. Check Small: can it be completed in one sprint? If not, split it
6. Check Testable: are acceptance criteria clear enough to write tests?
7. Rewrite any stories that fail 2+ INVEST criteria and re-review with the team

### Verification
- All stories evaluated against INVEST
- Failing stories rewritten
- Team confirms stories are sprint-ready

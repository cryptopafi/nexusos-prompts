# DPM-037 — Formulate Usability Hypothesis

## Problema
Teams ship features assuming they are usable, then wonder why adoption is low.

## Procedura
### Prerequisites
- Product feature or prototype
- User task definition
- Usability benchmarks

### Steps
1. Identify the key task users must complete with the feature
2. State the usability hypothesis: 'We believe [user] can complete [task] in [time] with [error rate] on first attempt'
3. Define usability metrics: task completion rate, time on task, error count, satisfaction score
4. Set success criteria: e.g., '80% completion rate, <3 minutes, <2 errors'
5. Design the usability test: 5 participants, think-aloud protocol, specific task scenarios
6. Prepare the test materials: prototype, task descriptions, observation template
7. Run the test and compare results to success criteria

### Verification
- Usability hypothesis stated with metrics
- Test designed and run
- Results compared to criteria
